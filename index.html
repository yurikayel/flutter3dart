🚀 Flutter 3.10 e Dart 3: O que há de novo? 🎉

O Flutter 3.10 e o Dart 3 foram lançados com uma série de recursos e melhorias emocionantes. Vamos dar uma olhada no que há de novo! 🏊‍♀️📚

Dart 3 🎯

O Dart 3 trouxe a segurança de nulos e recursos importantes para a linguagem. Agora, o Dart é uma linguagem 100% segura em relação a nulos, o que significa que você pode confiar completamente no sistema de tipos. 🙅‍♀️💼 [1, 3]

Além disso, o Dart 3 introduziu a visualização da compilação para WebAssembly (Wasm), permitindo que o Dart seja executado em diferentes ambientes compatíveis com WebAssembly. Isso abre um novo mundo de possibilidades para a execução do Dart em vários contextos. 🌐💻 [1]

Mudanças quebradas no Dart 3 🚧🔨

Com o Dart 3, algumas mudanças significativas foram introduzidas, exigindo ajustes no código existente:

  • Segurança de Nulos: O Dart 3 implementou uma abordagem mais rigorosa em relação à segurança de nulos, o que pode exigir ajustes no código existente para garantir a conformidade. É importante revisar o código e adotar as práticas recomendadas para a segurança de nulos. 🛠️🔒 [1]

  • Recursos da linguagem: O Dart 3 trouxe novos recursos importantes para a linguagem. Esses recursos podem afetar o código existente, exigindo atualizações e ajustes para aproveitar ao máximo as melhorias. É recomendado revisar a documentação oficial do Dart para entender completamente essas mudanças e adotar as melhores práticas. 📚💻 [1]

Flutter 3.10 🦋

O Flutter 3.10 trouxe uma série de melhorias e recursos para aprimorar o desenvolvimento de aplicativos multiplataforma. Vamos dar uma olhada no que mudou! 🍏💻

Mudanças quebradas no Flutter 3.10 🚧🔨

No Flutter 3.10, foram introduzidas algumas mudanças significativas que exigem atenção durante a atualização:

  • Aperfeiçoamentos no Material 3 widget toolkit: O Flutter 3.10 trouxe melhorias para o conjunto de ferramentas de widgets Material 3, permitindo criar interfaces de usuário ainda mais bonitas e ricas em recursos. Essas melhorias podem afetar a aparência e o comportamento dos aplicativos Flutter existentes, exigindo atualizações no código e nos estilos.

  • Suporte expandido para macOS e iOS: O Flutter 3.10 expandiu o suporte para as plataformas macOS e iOS, tornando o desenvolvimento de aplicativos nativos para dispositivos Apple ainda mais suave. Isso permite que os desenvolvedores criem aplicativos nativos bonitos e de alto desempenho para essas plataformas. 🍎📱 [2]

  • Renderizador padrão no iOS: O Flutter 3.10 adotou o renderizador Impeller como o renderizador padrão no iOS, substituindo o renderizador anterior. Essa mudança tem o objetivo de oferecer um desempenho mais consistente e uma experiência de usuário aprimorada para os aplicativos Flutter no iOS. 🌉📱 [6]

  • Nova ponte JNI para bibliotecas Jetpack: O Flutter 3.10 introduziu uma nova ponte JNI que permite a chamada direta de bibliotecas Jetpack escritas em Kotlin a partir do Dart, sem a necessidade de um plugin externo. Isso facilita a integração de bibliotecas e recursos do ecossistema Jetpack em aplicativos Flutter. 🌉📚 [7]

  • Outras melhorias e correções de bugs: O Flutter 3.10 também traz diversas melhorias de desempenho, correções de bugs e otimizações gerais para tornar o desenvolvimento de aplicativos Flutter ainda mais eficiente e robusto. É recomendado revisar as notas de lançamento oficiais para obter mais detalhes sobre essas melhorias. ⚙️🐛 [2]

🚀 PreferredSizeWidget

O PreferredSizeWidget é um widget do Flutter usado para definir o tamanho preferido de outros widgets, como no caso do AppBar. Vamos dar uma olhada nas mudanças relacionadas a esse widget! 📏🔧

Mudanças Dart 3 🚧🔨

Com a introdução do Dart 3, houve mudanças nas regras em torno do uso de mixins, o que afetou o uso do PreferredSizeWidget como mixin. Agora, é necessário utilizar a palavra-chave “implements” em vez de “with” ao usar o PreferredSizeWidget. Por exemplo:

Antes (versões anteriores ao Dart 3):

class MyAppBar extends StatelessWidget with PreferredSizeWidget {
  // ...
}

Depois (Dart 3 e posterior):

class MyAppBar extends StatelessWidget implements PreferredSizeWidget {
  // ...
}

Essa alteração torna as regras dos mixins mais rigorosas no Dart 3. É importante atualizar o código existente de acordo para garantir a conformidade com a nova abordagem. 🛠️🔒 [1][4]

Uso 🦋

O PreferredSizeWidget é comumente usado no Flutter para definir o tamanho preferido do AppBar. Ele permite que você adicione um widget na parte inferior do AppBar, como um TabBar, por exemplo. Apenas widgets que implementam o PreferredSizeWidget podem ser usados nessa posição. 📚📱 [2][5]
